## ✅ اصلاح ۱ — ساخت `XScraper` در هر سیکل (رفع باگ انجماد وضعیت)

**مشکل قبلی:** در `scheduler_loop` نمونهٔ `XScraper` یک‌بار و **بیرون** از حلقهٔ
`while True` ساخته می‌شد. چون اعتبارسنجی نشست فقط در `__init__` انجام می‌شود، مقدار
`mock_mode` در لحظهٔ startup منجمد می‌ماند:
- اگر نشست بعداً منقضی می‌شد، هشدار «کوکی منقضی» هیچ‌وقت ارسال نمی‌شد.
- اگر در startup کوکی خراب بود و ادمین بعداً آن را درست می‌کرد، ربات همچنان توییت‌های
  جعلی (Mock) تولید می‌کرد.

**اصلاح اعمال‌شده (`bot.py`):** خط ساخت `XScraper` از بیرون حلقه به **داخل بلوک `try`
درون حلقه** منتقل شد. حالا در هر سیکل یک نمونهٔ تازه ساخته و اعتبارسنجی می‌شود، و همان
نمونه در طول همان سیکل بازاستفاده می‌گردد (هم برای چک کوکی و هم برای ingestion از طریق
پارامتر `x_scraper`).

```python
# قبل:
def scheduler_loop():
    ...
    x_scraper = scraper.XScraper()   # یک‌بار در کل عمر برنامه
    while True:
        try:
            if config.X_USERNAME:
                if x_scraper.mock_mode: ...

# بعد:
def scheduler_loop():
    ...
    while True:
        try:
            # نمونهٔ تازه در هر سیکل → اعتبارسنجی مجدد و mock_mode به‌روز
            x_scraper = scraper.XScraper()
            if config.X_USERNAME:
                if x_scraper.mock_mode: ...
            ...
            scraper.run_scraper_ingestion(x_scraper=x_scraper)
```

**نتیجه:** هم تک‌بودن نمونه در هر سیکل حفظ شد (هدف ر۱-۲)، هم اعتبارسنجی تازه در هر دور
بازگشت (هدف ۲-۱/ر۱-۳). دیگر `mock_mode` منجمد نمی‌ماند و هشدار کوکی به‌درستی کار می‌کند.

## ✅ اصلاح ۲ — مستندسازی وابستگی twifork ↔ twikit (رفع ر۱-۹)

**مشکل قبلی:** کد `from twikit import Client` را ایمپورت می‌کرد ولی `requirements.txt`
بستهٔ `twifork` را نصب می‌کرد، بدون هیچ توضیحی. این جفت‌شدگی برای توسعه‌دهندهٔ بعدی گیج‌کننده
و شکننده بود.

**اصلاح اعمال‌شده:** یک فایل `README.md` ساخته شد که علاوه بر راهنمای راه‌اندازی، بخش
مخصوصی دارد با عنوان «نکتهٔ مهم دربارهٔ وابستگی X/Twitter» که توضیح می‌دهد:
- بستهٔ `twifork` با نام پکیج `twikit` نصب می‌شود، پس ایمپورت `twikit` درست است.
- این جفت‌شدگی عمدی است و نباید `twikit` رسمی را جداگانه نصب کرد (خطر تداخل).
- نصب فقط باید از طریق `requirements.txt` انجام شود.

README همچنین ساختار فایل‌ها، متغیرهای محیطی و ثابت‌های قابل‌تنظیم `config.py` را مستند
می‌کند.
